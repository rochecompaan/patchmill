# Linting and Pre-commit Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> superpowers:subagent-driven-development (recommended) or
> superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add TypeScript linting, Markdown linting, Prettier formatting with
80-character Markdown prose wrapping, and a Husky/lint-staged pre-commit hook.

**Architecture:** Keep tooling configuration in focused root-level config files.
Add npm scripts for full-repo checks and formatting, and use lint-staged so
pre-commit formats and checks only staged files.

**Tech Stack:** Node 24, npm, ESLint flat config, `typescript-eslint`, Prettier,
`markdownlint-cli2`, Husky, lint-staged.

---

## File map

- Create: `eslint.config.js` — flat ESLint config for TypeScript files in
  `bin/`, `scripts/`, `src/`, and `test-support/`.
- Create: `.markdownlint-cli2.jsonc` — Markdown lint config for `README.md`,
  `docs/**/*.md`, and bundled skill Markdown files.
- Create: `.prettierrc.json` — Prettier config with `printWidth: 80` and
  `proseWrap: "always"`.
- Create: `.prettierignore` — ignore dependency, generated, and local state
  directories.
- Create: `.husky/pre-commit` — pre-commit hook that runs lint-staged.
- Modify: `package.json` — add lint/format scripts, dev dependencies, `prepare`,
  and lint-staged config.
- Create: `package-lock.json` — npm lockfile generated by dependency
  installation.


Expected: npm updates `package.json` and creates `package-lock.json`.

- [ ] **Step 3: Add npm scripts and lint-staged config**

Update `package.json` so the `scripts` object includes these entries while
preserving existing scripts:

```json
{
  "lint": "npm run format:check && npm run lint:ts && npm run lint:md",
  "lint:ts": "eslint \"{bin,scripts,src,test-support}/**/*.ts\" --max-warnings=0",
  "lint:md": "markdownlint-cli2",
  "format": "prettier --write .",
  "format:check": "prettier --check .",
  "prepare": "husky"
}
```

Add this top-level `lint-staged` config to `package.json`:

```json
{
  "lint-staged": {
    "*.{ts,js,json,md}": "prettier --write",
    "*.ts": "eslint --max-warnings=0",
    "*.md": "markdownlint-cli2"
  }
}
```

- [ ] **Step 4: Run dependency metadata check**

Run:

```sh
npm install
```

Expected: npm exits 0 and keeps `package-lock.json` consistent with
`package.json`.

## Task 2: Add TypeScript ESLint configuration

**Files:**

- Create: `eslint.config.js`

- [ ] **Step 1: Run TypeScript lint before config exists**

Run:

```sh
npm run lint:ts
```

Expected: ESLint exits non-zero because `eslint.config.js` does not exist yet.

- [ ] **Step 2: Create ESLint flat config**

Create `eslint.config.js`:

```js
import js from "@eslint/js";
import globals from "globals";
import tseslint from "typescript-eslint";

export default tseslint.config(
  {
    ignores: [
      "coverage/**",
      "node_modules/**",
      ".devenv/**",
      ".direnv/**",
      ".patchmill/**",
      ".pi/**",
      ".tmp/**",
      ".worktrees/**",
    ],
  },
  js.configs.recommended,
  ...tseslint.configs.recommended,
  {
    files: ["**/*.ts"],
    languageOptions: {
      globals: {
        ...globals.node,
      },
      parserOptions: {
        sourceType: "module",
      },
    },
    rules: {
      "@typescript-eslint/no-explicit-any": "off",
      "@typescript-eslint/no-unused-vars": [
        "error",
        {
          argsIgnorePattern: "^_",
          caughtErrorsIgnorePattern: "^_",
          varsIgnorePattern: "^_",
        },
      ],
    },
  },
);
```

- [ ] **Step 3: Run TypeScript lint**

Run:

```sh
npm run lint:ts
```

Expected: ESLint runs against TypeScript files. If it reports genuine issues,
make minimal code changes or narrowly adjust config only when a rule is not
useful for this repo.
