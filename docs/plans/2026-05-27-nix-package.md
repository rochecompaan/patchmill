# Nix Package Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> superpowers:subagent-driven-development (recommended) or
> superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a reproducible Nix package for Patchmill that exposes `patchmill`
through flake package outputs and a reusable derivation file.

**Architecture:** `nix/package.nix` owns the Node package derivation and can be
imported by downstream Nix configurations. `flake.nix` imports that derivation
for each supported system and exposes it as both `packages.${system}.patchmill`
and `packages.${system}.default`. Verification builds the flake output and
smoke-tests the installed CLI.

**Tech Stack:** Nix flakes, `pkgs.buildNpmPackage`, Node.js 24, npm lockfile
dependency vendoring.

---

## File structure

- Create `nix/package.nix`: focused derivation for building and installing the
  Patchmill Node CLI.
- Create `flake.nix`: flake metadata and package outputs for common Linux and
  Darwin systems.
- Create `flake.lock`: generated by `nix flake lock` / `nix build`, pinning
  `nixpkgs`.
- Modify `nix/package.nix`: replace the fake npm dependency hash with the real
  hash reported by Nix.

## Task 1: Add the reusable package derivation

**Files:**

- Create: `nix/package.nix`

- [ ] **Step 1: Create `nix/package.nix` with a fake npm dependency hash**

Write this file exactly:

```nix
{
  lib,
  buildNpmPackage,
  nodejs_24,
  makeWrapper,
}:

let
  buildNpmPackageNode24 = buildNpmPackage.override { nodejs = nodejs_24; };
in
buildNpmPackageNode24 rec {
  pname = "patchmill";
  version = "0.0.0";

  src = lib.cleanSourceWith {
    src = lib.cleanSource ../.;
    filter = path: type:
      let
        baseName = baseNameOf path;
      in
      !(baseName == ".git"
        || baseName == "node_modules"
        || baseName == ".devenv"
        || baseName == ".patchmill"
        || baseName == "result");
  };

  npmDepsHash = lib.fakeHash;

  dontNpmBuild = true;

  nativeBuildInputs = [ makeWrapper ];

  env = {
    HUSKY = "0";
  };

  doCheck = true;
  checkPhase = ''
    runHook preCheck
    npm test
    runHook postCheck
  '';

  postInstall = ''
    rm -f "$out/bin/patchmill"
    makeWrapper ${nodejs_24}/bin/node "$out/bin/patchmill" \
      --add-flags "$out/lib/node_modules/${pname}/bin/patchmill.ts"
  '';

  meta = {
    description = "Agent-driven software factory that turns issues into reviewed diffs";
    mainProgram = "patchmill";
    license = lib.licenses.mit;
    platforms = lib.platforms.unix;
  };
}
```

## Task 2: Add flake package outputs

**Files:**

- Create: `flake.nix`
- Create: `flake.lock`

- [ ] **Step 1: Create `flake.nix`**

Write this file exactly:

```nix
{
  description = "Patchmill - an agent-driven software factory";

  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";
  };

  outputs =
    { nixpkgs, ... }:
    let
      systems = [
        "x86_64-linux"
        "aarch64-linux"
        "x86_64-darwin"
        "aarch64-darwin"
      ];

      forAllSystems = nixpkgs.lib.genAttrs systems;
    in
    {
      packages = forAllSystems (
        system:
        let
          pkgs = import nixpkgs { inherit system; };
          patchmill = pkgs.callPackage ./nix/package.nix { };
        in
        {
          inherit patchmill;
          default = patchmill;
        }
      );
    };
}
```

- [ ] **Step 2: Generate `flake.lock`**

Run:

```bash
nix flake lock
```

Expected: exit code `0`; `flake.lock` is created and pins `nixpkgs`.

- [ ] **Step 3: Verify the package output is discoverable**

Expected: output lists `packages.x86_64-linux.default` and
`packages.x86_64-linux.patchmill` on Linux, plus equivalent outputs for the
