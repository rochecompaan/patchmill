# Nix package design

## Goal

Add an idiomatic, reproducible Nix package for Patchmill that can be consumed
directly from this repository or imported by another Nix configuration.

## Chosen approach

Use both a reusable package file and flake outputs:

- `nix/package.nix` defines the Patchmill derivation.
- `flake.nix` exposes the derivation as `packages.${system}.patchmill` and
  `packages.${system}.default`.

This keeps packaging logic reusable while making the repository easy to build
with `nix build`.

## Package implementation

The derivation will use `pkgs.buildNpmPackage` because Patchmill is a Node CLI
with a committed `package-lock.json` and runtime dependency on `pi-subagents`.

Expected characteristics:

- Build with Node.js 24 to match `package.json` engines and `devenv.nix`.
- Use the repository root as source so `package.json`, `package-lock.json`,
  `bin/`, `src/`, and docs are available.
- Use the normal Nix npm dependency hash cycle: start with `pkgs.lib.fakeHash`,
  run the build, and replace it with the reported hash.
- Avoid running `prepare` hooks during packaging because Husky is
  development-only.
- Run the existing test suite in `checkPhase` where feasible.
- Install a `patchmill` executable under `$out/bin`.

## Flake implementation

`flake.nix` will:

- Pin `nixpkgs` through the flake lock generated by Nix.
- Support common Linux and Darwin systems without pulling in a separate
  flake-utils dependency.
- Import `nix/package.nix` with the appropriate `pkgs` set.
- Expose both named and default package outputs.

## Verification

Fresh verification will include:

- `nix build .#patchmill`
- `./result/bin/patchmill --help`

If Nix reports a required `npmDepsHash`, update the derivation and rerun the
build. If tests fail during the Nix build, investigate the failure before
changing package behavior.

## Scope boundaries

This change does not alter Patchmill CLI behavior, repository workflow
semantics, or project configuration loading. It only adds Nix packaging and the
minimal metadata required to build and run the existing CLI from Nix.
